METTLE

Updated Design Document
Ryan Goldberg | March 13, 2026 | Living Document | v4

Mettle has moved from a concept pitch into a working product prototype. This v4 document replaces the older v3 design doc as the current source-of-truth summary for the repo at /Users/ryan/Documents/GitHub/MettleOS. It preserves the original core vision while updating the design around what is now implemented, what has changed, and what still needs to be built.

Short version: Mettle is still the custom Old School RuneScape challenge skill built around personalized writs, public accountability, and a path to level 99. The big difference is that Mettle now has a live Next.js prototype with real draft logic, a ledger UI, Trials, Forks, Landmarks, debt pressure, reckoning pressure, local persistence, and a new seal economy.

1. The Current Concept

Mettle is a custom progression skill layered on top of Old School RuneScape. It exists in two forms at the same time:

  • A creator format for YouTube and streaming, where the account is pushed into neglected or uncomfortable content.
  • A playable web tool that lets other players run their own Mettle ledger from their real account data.

The core idea is still intact: the account already reveals what the player has avoided. Mettle reads that pattern, turns it into named writs, and makes the player answer for it over time.

The biggest terminology update is that the main surface is now called the ledger rather than the board. “Writ” remains the name for the challenge unit itself.

2. What Changed Since v3

  • The project is now a functioning app prototype instead of only a design plan.
  • The UI surface evolved from “writ board” language toward a more system-driven “ledger” identity.
  • The account model now tracks 24 skills and 69 bosses, including newer OSRS content and a Sailing placeholder.
  • Trial selection is more account-aware than the original fixed milestone pitch. Several Trials now resolve dynamically from untouched bosses or low-KC targets.
  • The draft system gained more states: Normal, Cursed, Favored, Hot Streak, and Final Trial.
  • Mettle now includes a live seal economy: reroll a draft, buy an extra draft choice, or remove a modifier from an active writ.
  • Debt and reckoning cleanup now intentionally pay reduced rewards: half XP, no seals, and no streak gain.
  • The current build uses local browser persistence, not Supabase or server-side run storage.
  • Quest Cape still exists as a Landmark, but because Wise Old Man cannot fully verify quest completion from the current integration, it is triggered manually.
  • The prototype already has an improved mobile pass and a more intentional visual identity than the original MVP framing.

3. Current Product Experience

Player Flow

StepCurrent Behavior
1. EntryPlayer lands on a branded cold-open screen and chooses Wise Old Man lookup or manual stat entry.
2. IntakeThe app fetches skills and boss KC through a local API route that refreshes Wise Old Man data, or accepts manual values.
3. Run StartThe player enters the ledger with Mettle XP, seals, streak, debt, history, and draft systems active.
4. ProgressionThe player draws writs, chooses one, resolves it, and accumulates progress toward higher tiers and eventual path assignment.
5. Return VisitsRuns reopen automatically from local storage, so the prototype supports an ongoing single-player save loop without account auth.

Current Surface Areas

  • Ledger: active writ, draft state, debt queue, reckoning queue, trial reveals, fork presentations, and landmark moments.
  • Stats: skill audit, boss ledger, system state, path information, reckoning tracker, fork history, landmark tracker.
  • History: archived resolved writs with results, XP earned, seals earned, modifiers, timestamps, and expansion details.

5. Generator Logic and Account Modeling

Inputs

  • RuneScape name through Wise Old Man API
  • Manual fallback for skills and boss KC
  • All 24 tracked skills
  • 69 tracked bosses

The local API route first asks Wise Old Man to refresh a player profile, waits briefly, then fetches the latest snapshot. This makes the app feel more live than a static account import.

Gap Logic

The original account-average idea still drives the system, but the live prototype is more nuanced than v3. Instead of relying mainly on fixed “gain 5 levels” style tasks, many skill writs now scale around the account’s current weakest skills, level bands, and practical target ranges. At higher levels, some writs convert into chunk-based progression asks instead of always demanding a full jump to account average.

Weighting Factors

  • Tier eligibility
  • Completion status and repeatability
  • Recent draft suppression
  • Skill gap severity
  • Combat average
  • Boss touch count and total boss KC
  • Low-KC or zero-KC bonuses for relevant bosses
  • Fresh-account bias toward PvM introductions rather than endurance bossing

This matters because the live Mettle system is no longer just a weighted random pick from a static list. It behaves more like an account-reading pressure engine.

Examples of Dynamic Behavior

  • Level 10 Trial can choose from Obor, Bryophyta, Scurrius, or Giant Mole based on account state.
  • Later Trials can pivot toward untouched God Wars bosses, lowest-KC raid-adjacent encounters, or the hardest untouched endgame boss.
  • Skill Gap writs frequently rewrite their own title and objective around the player’s actual weakest skill, lowest production skill, or lowest set of skilling stats.
  • Final path assignment is computed from what the player actually resolved over the course of the run.

7. Visual and UX Direction

The original doc correctly identified presentation as a core differentiator, and the live prototype already leans into that. The current app uses a dark, ceremonial, OSRS-adjacent presentation with strong gold accents, high-contrast interface panels, branded reveal states, and a more premium “ritual ledger” tone than a plain utility tool.

Current Visual Language

  • Cinematic entry screen instead of a plain form
  • Ledger framing rather than spreadsheet framing
  • Dedicated reveal moments for Trials, Forks, and Landmarks
  • Separate board, stats, and archive views
  • Responsive CSS pass for mobile layouts

The visual direction is stronger than the original MVP recommendation. The app is already trying to feel like part game interface, part archive, and part ceremonial challenge system.

9. What Is Live vs. What Is Still Missing

Live Now

  • Branded entry screen
  • Wise Old Man account intake
  • Manual stat and KC fallback
  • Core draft loop
  • Debt and cursed states
  • Reckoning system
  • Trials, Forks, Landmarks, and Final Trial paths
  • Seal economy
  • Run history and archive
  • Mobile-responsive pass
  • Local save / resume loop

Still Missing or Intentionally Deferred

  • Cloud persistence and login
  • Shareable public Mettle profiles
  • Social comparison features
  • Quest verification beyond manual Quest Cape confirmation
  • Admin tooling for balancing and content authoring
  • Series-brand asset package: logo system, typography, channel packaging, and full motion toolkit
  • RuneLite integration or automatic account resync beyond current Wise Old Man pulls

Important Reality Check

Some design ideas from v3 remain part of the project identity but are not yet hard-coded as isolated systems. For example, wilderness pressure is present in content and tier framing, but not as a dedicated wilderness-only rules engine. The format is clearly alive; some edge-rule specificity is still better described in content than in code.

10. Recommended Next Phase

If the goal is to move from strong prototype to launch-ready product and creator platform, the next phase should focus on six priorities:

  1. Persistent accounts and shareable profiles. This is the biggest product multiplier after the current prototype.
  2. Content authoring tools. The writ pool is already large enough that editing raw data files will become painful over time.
  3. Balance instrumentation. Add analytics or debug views so the team can tune draft weights, defers, seals, and Trials with evidence instead of feel alone.
  4. Quest and progression verification. Reduce manual confirmation wherever Wise Old Man or future integrations make that possible.
  5. Brand package and series tooling. Bring the logo, crest states, overlays, and episode graphics up to the same level as the product idea.
  6. Public rollout planning. Decide whether Mettle remains mostly a creator-led format first or whether the public tool should be prepared for broader adoption early.

11. Summary

v3 captured the right vision. v4 documents the fact that Mettle is now a real system with a real UI, a real account model, and a real gameplay loop. The project is no longer “should this work?” It is now “how polished, social, and scalable should this become?”

Mettle’s identity is strongest where the original concept and the current implementation overlap: personalized challenge pressure, visible avoidance, ceremonial progression, and a finale that feels earned. The job from here is not inventing the product. It is sharpening it.
