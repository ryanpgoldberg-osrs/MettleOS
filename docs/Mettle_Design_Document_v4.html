1. Current Position

Mettle is still a custom progression skill layered on top of Old School RuneScape. It still serves two purposes at once:

  • A creator format built around visible avoidance, escalating pressure, and an ascent to level 99.
  • A player-facing web product that generates account-specific tasks from real account state.

The core idea has not changed: the account itself exposes what the player has avoided. Mettle reads that pattern, turns it into directed pressure, and tracks the answer over time.

The product language is now intentionally aligned around task terminology, while ledger remains the name for the broader ascent state and progression surface.

2. What Changed Recently

  • The task pool has expanded materially, especially in Quest and Exploration content.
  • The live public deployment now sits at https://mettle.quest/.
  • Mettle now supports full account sync import, quest-only sync import, and whole-ascent save import/export.
  • Quest state and achievement diary state are now first-class parts of the ascent model.
  • The draft generator is now quest-aware and can suppress objectives already satisfied by synced progress.
  • Quest Cape can now be detected from synced quest data.
  • Forks can react to imported quest or boss progress, including auto-resolution when both branches are already complete.
  • An accusation system now escalates repeated avoidance into explicit Judgment or Defense tasks.
  • Seals can now clear a deferred task instantly.
  • The project now includes a dedicated RuneLite plugin workspace, sync schemas, and a local development launcher flow.
  • The web app now includes Vercel Analytics and Speed Insights.
  • Mettle now includes an optional built-in merchant utility for GE flipping.

Current Web-App Sync Flow

  1. Import a Mettle account sync JSON file from the entry screen or from an existing ascent.
  2. Normalize skills, quests, diaries, and sync metadata locally.
  3. Refresh Wise Old Man using the same RSN to enrich boss KC and merge in any higher public skill levels when available.
  4. Save the merged state into the browser ascent.

The plugin is becoming the best source for private progression state like quests and diaries, while Wise Old Man remains the easiest source for public stat and boss history. The app is now deliberately built to combine those two surfaces, and account-sync import still succeeds even if the Wise Old Man enrichment pass is temporarily unavailable.

Validation APIs

  • /api/player/[rsn] triggers a Wise Old Man refresh and returns a normalized player payload for the web app.
  • /api/account-sync validates and normalizes full account sync payloads.
  • /api/quest-sync validates and normalizes quest sync payloads.

Both routes are validation-only today. They do not persist sync data because Mettle still runs on local browser saves rather than linked user accounts or server-side sessions.

6. Generator And Rule Logic

Core Inputs

  • 24 tracked skills
  • 69 tracked bosses
  • quest completion data
  • quest point totals
  • Quest Cape state
  • achievement diary summaries
  • recent draft history, debt state, streak state, favored state, and completed task history

Quest-Aware Task Filtering

The generator no longer treats quest progress as invisible. It now has rule-level awareness of specific completed quests, quest point targets, Quest Cape completion, and boss objectives already satisfied by synced boss KC.

That moves Mettle away from “interesting flavor around a public stats import” and toward a real account-state rules engine.

Weighting Model

The weighting model still considers the same major account signals:

  • tier eligibility
  • repeatability and completion state
  • recent draft suppression
  • skill gap severity
  • combat average
  • boss touch count
  • total boss KC
  • low-KC and zero-KC bonuses

The difference now is that synced quest and boss completion data can remove already-cleared objectives before weighting even starts.

The merchant desk is not part of the Mettle ascent model. It is a quality-of-life utility that lives in the same product so it can share the same deployment and brand surface without contaminating core progression logic.

8. RuneLite Plugin Project

Mettle now includes a separate RuneLite plugin project under runelite-plugin/. This is a real workspace with build files, source code, a sidebar export panel, and a local dev-run flow.

Current Plugin Scope

  • plugin entrypoint and config
  • sidebar panel for manual export
  • quest completion collection
  • achievement diary tier collection
  • stable account sync snapshot assembly
  • Gradle wrapper and local RuneLite development launcher flow

Boss KC is still the weakest part of the plugin story. The current player flow is still to let the web app enrich boss killcounts through Wise Old Man after import. In practice, the plugin owns private progression state while Wise Old Man still helps fill the public combat-history layer.

9. Visual, Product, And Deployment Direction

The visual direction remains one of Mettle's strongest differentiators. The app still aims to feel like a ceremonial ledger rather than a generic utility page. That direction is now reinforced by more product-like behavior:

  • the public deployment now lives at https://mettle.quest/
  • entry screen now explicitly supports account sync import
  • stats view includes synced account-state summaries
  • stats and history now expose accusation-state pressure instead of leaving that layer invisible
  • history records seals gained and seals spent, not just XP
  • the app shell includes Vercel Analytics and Speed Insights
  • the merchant desk lives inside the same UI instead of splintering into a separate tool

12. Recommended Next Phase

  1. Move from file import to linked sync. The plugin and contract work are far enough along that the next major milestone should be an authenticated or session-linked sync target.
  2. Add server persistence. The local-save model has gone as far as it can before social sharing, cross-device continuity, and direct plugin upload are blocked.
  3. Promote diary and quest sync into more task rules. The data is already in the model; the next value comes from using more of it in generation.
  4. Build balancing tooling. The task pool is now large enough that tuning by feel alone is not ideal.
  5. Keep product surfaces aligned. Continue treating the merchant desk, sync system, and ledger as one deployment without letting secondary tools muddy the core ascent loop.

13. Summary

Mettle is still the same product at its core: a personalized challenge system that turns account neglect into visible pressure. The difference is that the project now has a real second layer of infrastructure underneath it. It is no longer just a stylish prototype with weighted drafts. It now has import/export contracts, richer saved state, accusation logic, a plugin companion, a merchant utility, and sync-aware rules.

The center of gravity has shifted from “can this be a compelling format?” to “how quickly can the app, sync model, judgment layer, creator package, and side utilities be brought into one polished system?” That is the current stage of the project.
