METTLE

Updated Design Document
Ryan Goldberg | March 14, 2026 | Living Document | v4

This file is the current source-of-truth design summary for the repo at /Users/ryan/Documents/GitHub/MettleOS. It replaces the earlier v4 draft by folding in everything added since then: synced quest and diary state, import/export flows, account sync contracts, a RuneLite plugin project, quest-aware drafting, seal-based deferred-task clearing, and deployment instrumentation.

Current state: Mettle is no longer just a Wise Old Man plus manual-entry prototype. It is now a local-save web app with multiple intake paths, quest-aware progression rules, save import/export, sync validation APIs, and a parallel RuneLite plugin effort that is becoming the long-term default account import path.

1. Current Position

Mettle is still a custom progression skill layered on top of Old School RuneScape. It still serves two purposes at once:

  • A creator format built around visible avoidance, escalating pressure, and a run to level 99.
  • A player-facing web product that generates account-specific tasks from real account state.

The core idea has not changed: the account itself exposes what the player has avoided. Mettle reads that pattern, turns it into directed pressure, and tracks the answer over time.

The product language is now intentional and consistent. The content model, live UI, and save formats all use task terminology, while ledger remains the name for the broader run state and progression framing.

2. What Changed Since the Last Version

  • Mettle now supports full account sync import in addition to Wise Old Man and manual entry.
  • Quest state and achievement diary state are now first-class parts of the saved run.
  • The app can now import and export Mettle save files, with versioned save migration logic.
  • The draft generator is now quest-aware and can suppress objectives already satisfied by synced quest or boss progress.
  • Quest Cape is no longer only a manual late-game placeholder. It can now be detected from synced quest data.
  • Forks can now react to imported progress, including auto-resolving when both branches are already complete.
  • Seals gained a new use: clear a deferred task instantly without earning recovery XP.
  • The project now includes a dedicated RuneLite plugin workspace, sync schemas, build tooling, and a local dev launcher flow.
  • The web app now has Vercel Analytics and Vercel Speed Insights wired into the app shell.

Quest Cape has materially changed since the last draft of this doc. It is no longer just a manual workaround for Wise Old Man limits. If quest sync data says the cape is earned, the Landmark can now trigger from synced state. The manual button still matters as a fallback, but it is no longer the primary long-term plan.

Fork handling also changed in a meaningful way. When synced quest or boss data shows that a fork branch is already complete, the fork can narrow itself. If both sides are already complete, the fork auto-resolves instead of pretending the choice still exists.

Current Web-App Sync Flow

  1. Import a Mettle account sync JSON file from the entry screen or inside an existing run.
  2. Parse and normalize skills, quests, diaries, and sync metadata locally.
  3. Refresh Wise Old Man using the same RSN to enrich boss KC if possible.
  4. Save the merged state into the browser run.

The important architectural truth is that the plugin is becoming the best source for private progression state like quests and diaries, while Wise Old Man remains the easiest source for public stat and boss data. The current app is explicitly built to combine those two surfaces.

Validation APIs

  • /api/account-sync validates and normalizes full account sync payloads.
  • /api/quest-sync validates and normalizes quest sync payloads.

Both routes are validation-only right now. They do not persist sync data because the app still runs on local browser saves rather than linked user accounts or server-side sessions.

6. Generator and Rule Logic

Core Inputs

  • 24 tracked skills
  • 69 tracked bosses
  • Quest completion data
  • Quest point totals
  • Quest Cape state
  • Achievement diary summaries
  • Recent draft history, debt state, streak state, favored state, and completed task history

Quest-Aware Task Filtering

This is one of the biggest design changes since the last update. The generator no longer treats quest progress as invisible. It now has rule-level awareness of:

  • specific completed quests
  • quest point targets
  • Quest Cape completion
  • boss objectives already satisfied by synced boss KC

That means Mettle is moving away from “interesting flavor around a public stats import” and toward a real account-state rules engine.

Weighting Model

The weighting model still considers the same major account signals:

  • tier eligibility
  • repeatability and completion state
  • recent draft suppression
  • skill gap severity
  • combat average
  • boss touch count
  • total boss KC
  • low-KC and zero-KC bonuses

The difference now is that synced quest and boss completion data can remove already-cleared objectives before weighting even starts.

Dynamic Behavior That Matters Now

  • Dynamic Trials still pick from account state instead of one hard-coded ask wherever possible.
  • Quest-related tasks can be suppressed automatically if imported quest sync says they are already done.
  • Fork options can be marked unavailable or auto-resolved based on synced progress.
  • Quest Cape Landmarks can trigger from synced completion instead of relying only on manual input.

8. RuneLite Plugin Project

Mettle now includes a separate RuneLite plugin project under runelite-plugin/. This is no longer just an idea in notes. It is a real project with build files, source code, a local dev flow, and an export panel.

Current Plugin Scope

  • plugin entrypoint and configuration
  • sidebar panel for manual export
  • quest completion collection
  • achievement diary tier collection
  • stable account sync snapshot assembly
  • Gradle wrapper and local RuneLite development launcher flow

Important Architecture Note

Boss KC is still the weak link in the plugin story. The plugin contract includes boss data, but the current intended player flow is to let the web app re-fetch or enrich boss killcounts through Wise Old Man after import. In practice, the plugin owns the private progression layer while Wise Old Man still helps fill the public combat history layer.

Contract and Repo Layout Added Since Last Update

  • runelite-plugin/ - dedicated plugin workspace
  • sync-contract/ - canonical account sync schema and example payload
  • docs/mettle-account-sync-plan.md - rollout and architecture notes
  • docs/mettle-quest-sync-plan.md - quest sync bridge notes

The project is now clearly split into two collaborating surfaces: the web app and the RuneLite companion.

9. Visual, Product, and Deployment Direction

The visual direction remains one of Mettle's strongest differentiators. The app still aims to feel like a dark ceremonial ledger rather than a generic utility page. That direction is now reinforced by stronger operational decisions too:

  • entry screen now explicitly supports account sync import, not just stat lookup
  • stats view now includes synced account-state summaries
  • history view now records seals gained and seals spent, not just XP
  • the app shell now includes Vercel Analytics and Speed Insights

This is a useful shift. Mettle is no longer only trying to look premium; it is beginning to behave like a deployable product with instrumentation.

12. Recommended Next Phase

  1. Move from file import to linked sync. The plugin and contract work are far enough along that the next major milestone should be an authenticated or session-linked sync target.
  2. Add server persistence. The local-save model has gone as far as it can before social sharing, cross-device continuity, and direct plugin upload are blocked.
  3. Promote diary and quest sync into more task rules. The data is now present; the next value comes from using more of it in generation.
  4. Build balancing tooling. The content pool is large enough that tuning by feel alone is no longer ideal.
  5. Keep terminology unified. Stay anchored on “task” language so future UI, docs, and save changes do not drift back into mixed naming.

13. Summary

Mettle is still the same product at its core: a personalized challenge system that turns account neglect into visible pressure. The difference is that the project now has a real second layer of infrastructure underneath it. It is no longer just a stylish prototype with weighted drafts. It now has import/export contracts, richer saved state, a plugin companion, and sync-aware rules.

The project's center of gravity has shifted from “can this be a compelling format?” to “how quickly can the app, sync model, and creator package be brought into one polished system?” That is a good problem to have.
